java中目前有四种解析xml文件的解析方式：DOM、SAX、JDOM、DOM4J

解析方式分析

基础方法：

  • DOM: 平台无关的官方解析方式
  • SAX: 基于事件驱动的解析方式

扩展方法（只有java平台可以使用）：

  • JDOM：
  • DOM4J：

DOM

DOM解析时，一次性将xml文件全部加入内存

优点

  • 树结构，直观，容易编写
  • 树结构保存在内存中，便于修改

缺点

  • 内存消耗大，影响解析性能，内存容易溢出

SAX

逐标签解析

优点

  • 事件驱动，内存消耗小
  • 仅适用于处理xml数据

缺点

  • 不易编码
  • 难以同时访问多处不同数据

JDOM

  • 使用具体类而不用接口
  • API大量使用Collections类

DOM4J

  • 使用接口和抽象基本方法
  • JDOM智能分支
  • 性能优异、灵活性好

具体性能分析

根据文件大小不同进行性能分析，对同一文件进行完全解析的时间进行比较（单位ms）

xml大小为 1kb

DOM性能：43
SAX性能：6
JDOM性能：48
DOM4J性能：45

xml大小为 100kb

DOM性能：89
SAX性能：32
JDOM性能：109
DOM4J性能：74

xml大小为 600kb

DOM性能：115
SAX性能：54
JDOM性能：209
DOM4J性能：149

xml大小为 1MB

DOM性能：162
SAX性能：80
JDOM性能：205
DOM4J性能：131

xml大小为 5MB（测试结果不稳定）

性能测试结果1：
    DOM性能：693
    SAX性能：142
    JDOM性能：529
    DOM4J性能：453
性能测试结果2：
    DOM性能：772
    SAX性能：161
    JDOM性能：496
    DOM4J性能：546
性能测试结果3：
    DOM性能：674
    SAX性能：155
    JDOM性能：414
    DOM4J性能：506
性能测试结果4：
    DOM性能：677
    SAX性能：169
    JDOM性能：447
    DOM4J性能：438

综上，SAX对全文件的解析速度最快，JDOM的解析速度最慢，而DOM和DOM4J的解析性能取决于文件，当文件大小达到1MB以上时，DOM4J的分析性能优于DOM，因此根据文件大小和具体使用情况选择不同的解析方式。

当文件大小达到5MB时，性能测试结果浮动偏差较大，DOM、JDOM、DOM4J这三种解析方式的性能强弱处于不稳定状态，虽然每次测试的文件都是一样的，但是为什么会发生这种情况还不太了解。

解析方式选择

  1. 在网站开发中，如果发布一条公告使用xml文件格式，可以采用DOM解析。公告的内容较少，所以文件大小不大，并且对不同节点进行修改也较为方便。

  2. 选择性数据读取时，不适合选择SAX解析方式，如根据某一产品型号，获取其名称等。

  3. 对于复杂的xml文档适合采用DOM4J。

Hibernate的xml解析方式采用的是DOM4J
